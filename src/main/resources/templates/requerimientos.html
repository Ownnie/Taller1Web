~R E Q U E R I M I E N T O S~

Conoce cuales son los requerimientos funcionales y no funcionales de nuestro proyecto

Funcionales

El sistema debe permitir que los usuarios arrendadores:

  • Activen su cuenta mediante un enlace enviado por correo.
  • Autentiquen su cuenta.
  • Vean sus propiedades disponibles para arrendamiento.
  • Creen propiedades.
  • Editen sus propiedades.
  • Suban fotos de sus propiedades.
  • Desactiven propiedades.
  • Vean las solicitudes de arriendo.
  • Acepten solicitudes de arriendo.
  • Rechacen solicitudes de arriendo.
  • Califiquen a los arrendatarios.

No funcionales

Se debe operar teniendo en cuenta que:

  • El sistema debe poder ser visto y utilizado desde cualquier navegador, incluyendo Chrome, Safari, Edge y Firefox.
  • El sistema debe manejar hasta xxx usuarios simultáneos sin degradación del rendimiento.
  • La interfaz de usuario debe ser intuitiva y fácil de usar para cualquier usuario.
  • El sistema debe proporcionar mensajes de error claros y específicos cuando se ingresen datos incorrectos en los formularios.
  • El sistema debe estar disponible el 90% del tiempo.
  • El sistema debe ser compatible con las versiones actuales de navegadores como Chrome, Safari, Edge y Firefox.
  • El sistema debe cifrar las contraseñas almacenadas.
