Endpoints para el Usuario Arrendatario

1. Crear Cuenta de Usuario Arrendatario

Detalles Descripción
Método HTTP: POST
URL: /api/arrendatarios
Descripción: Permite a un usuario arrendatario crear una cuenta en el sistema.
Parámetros:
  • nombres (string, requerido)
  • apellidos (string, requerido)
  • correo (string, requerido)
  • teléfono (string, opcional)
  • contraseña (string, requerido)
Respuesta: Confirmación de la creación de la cuenta y un enlace para activar la cuenta.

2. Activar Cuenta de Usuario Arrendatario

Detalles Descripción
Método HTTP: GET
URL: /api/arrendatarios/activar/{token}
Descripción: Activa la cuenta de un usuario arrendatario mediante un token enviado por correo electrónico.
Parámetros:
  • token (string, requerido)
Respuesta: Confirmación de la activación de la cuenta.

3. Autenticar Usuario Arrendatario

Detalles Descripción
Método HTTP: POST
URL: /api/arrendatarios/auth
Descripción: Autentica a un usuario arrendatario y proporciona un token de acceso
Parámetros:
  • correo (string, requerido)
  • contraseña (string, requerido)
Respuesta: Token de acceso y detalles del usuario

4. Buscar Propiedades

Detalles Descripción
Método HTTP: GET
URL: /api/propiedades
Descripción: Permite a un usuario arrendatario buscar propiedades disponibles.
Parámetros:
  • nombre (string, opcional)
  • municipio (string, opcional)
  • cantidad_personas (int, opcional)
Respuesta: Listado de propiedades que coinciden con los criterios de búsqueda.

5. Solicitar Arrendamiento

Detalles Descripción
Método HTTP: POST
URL: /api/arrendamientos
Descripción: Permite a un usuario arrendatario solicitar el arrendamiento de una propiedad.
Parámetros:
  • propiedad_id (int, requerido)
  • fecha_inicio (date, requerido)
  • fecha_fin (date, requerido)
Respuesta: Confirmación de la solicitud de arrendamiento.

2. Activar Cuenta de Usuario Arrendador

Descripción: Activa la cuenta de un usuario arrendador mediante un token enviado por correo electrónico.
Parámetros:

3. Autenticar Usuario Arrendador

Detalles Descripción
Método HTTP: POST
URL: /api/arrendadores/auth

Parámetros:
  • correo (string, requerido)
  • contraseña (string, requerido)

4. Crear Propiedad

Detalles Descripción
Método HTTP: POST
URL: /api/propiedades
Descripción: Permite a un usuario arrendador crear una nueva propiedad para arrendar.
Parámetros:
  • nombre (string, requerido)
  • departamento (string, requerido)
  • municipio (string, requerido)
  • tipo_ingreso (string, requerido)
  • descripción (string, requerido)
  • cantidad_habitaciones (int, requerido)
  • cantidad_baños (int, requerido)
  • permite_mascotas (boolean, requerido)
  • tiene_piscina (boolean, requerido)
  • tiene_asador (boolean, requerido)
  • valor_noche (decimal, requerido)
Respuesta: Confirmación de la creación de la propiedad.

5. Editar Propiedad

Detalles Descripción
Método HTTP: PUT
URL: /api/propiedades/{id}
Descripción: Permite a un usuario arrendador editar una propiedad existente.
Parámetros:
  • id (int, requerido)
  • nombre (string, opcional)
  • departamento (string, opcional)
  • municipio (string, opcional)
  • tipo_ingreso (string, opcional)
  • descripción (string, opcional)
  • cantidad_habitaciones (int, opcional)
  • cantidad_baños (int, opcional)
  • permite_mascotas (boolean, opcional)
  • tiene_piscina (boolean, opcional)
  • tiene_asador (boolean, opcional)
  • valor_noche (decimal, opcional)
Respuesta: Confirmación de la actualización de la propiedad.

6. Ver Solicitudes de Arrendamiento

Detalles Descripción
Método HTTP: GET
URL: /api/arrendamientos/solicitudes
Descripción: Permite a un usuario arrendador ver las solicitudes de arrendamiento para sus propiedades.
Parámetros:
  • estado (string, opcional)
Respuesta: Listado de solicitudes de arrendamiento con su estado correspondiente.

7. Aceptar Solicitud de Arrendamiento

Detalles Descripción
Método HTTP: PUT
URL: /api/arrendamientos/aceptar/{id}
Descripción: Permite a un usuario arrendador aceptar una solicitud de arrendamiento.
Parámetros:
  • id (int, requerido)
Respuesta: Confirmación de la aceptación de la solicitud de arrendamiento.

8. Cancelar Solicitud de Arrendamiento

Detalles Descripción
Método HTTP: PUT
URL: /api/arrendamientos/cancelar/{id}
Descripción: Permite a un usuario arrendador cancelar una solicitud de arrendamiento.
Parámetros:
  • id (int, requerido)
Respuesta: Confirmación de la cancelación de la solicitud de arrendamiento.

9. Calificar Arrendatario

Detalles Descripción
Método HTTP: POST
URL: /api/calificaciones/arrendatario
Descripción: Permite a un usuario arrendador calificar a un arrendatario.
Parámetros:
  • arrendamiento_id (int, requerido)
  • calificación (int, requerido)
  • comentario (string, opcional)
Respuesta: Confirmación de la calificación del arrendatario.
