~P R U E B A S~

Conoce como probamos nuestro sistema

FrontEnd

Los casos de prueba para el front son los siguientes:

Identificador: FE-01-1

Nombre corto: Visualización de la página de inicio

Descripción: Verificar que la página de inicio se carga correctamente y muestra todos los elementos necesarios (logotipo, menú de navegación, contenido principal).

Fecha y autor de creación: [Fecha] - [Tu nombre o equipo]

Prerequisitos: Usuario no autenticado.

Estado del software: El servidor está en funcionamiento.

Datos de prueba:

  • Datos de entrada: Acceso a la página de inicio sin autenticación.
  • Datos de salida: Página de inicio correctamente cargada con todos los elementos visibles.

Pasos:

  1. Navegar a la URL de la página de inicio.
  2. Verificar que el logotipo esté visible en la parte superior.
  3. Comprobar que el menú de navegación está completo y accesible.
  4. Asegurarse de que el contenido principal se muestra correctamente.

Puntos de validación:

  • Verificar que todos los elementos visuales (logotipo, menú, contenido) estén presentes y correctamente posicionados.
  • Comprobar que no haya errores de carga o elementos faltantes.

BackEnd

Los casos de prueba para el back son los siguientes:

Identificador: BE-01-1

Nombre corto: Autenticación de usuario

Descripción: Verificar que el sistema autentica correctamente a los usuarios con credenciales válidas y rechaza credenciales inválidas.

Fecha y autor de creación: [Fecha] - [Tu nombre o equipo]

Prerequisitos: Usuario registrado en la base de datos.

Estado del software: El sistema de autenticación está operativo.

Datos de prueba:

  • Datos de entrada: Credenciales válidas e inválidas (correo y contraseña).
  • Datos de salida: Tokens de autenticación o mensajes de error.

Pasos:

  1. Enviar una solicitud de autenticación con credenciales válidas.
  2. Verificar que el sistema devuelva un token de autenticación válido.
  3. Enviar una solicitud de autenticación con credenciales inválidas.
  4. Verificar que el sistema devuelva un mensaje de error apropiado.

Puntos de validación:

  • Asegurarse de que los tokens de autenticación sean válidos y se generen correctamente.
  • Verificar que los mensajes de error sean claros y específicos para credenciales inválidas.

Identificador: FE-01-2

Nombre corto: Formulario de registro funcional

Descripción: Asegurarse de que el formulario de registro funciona correctamente y valida la entrada del usuario.

Fecha y autor de creación: [Fecha] - [Tu nombre o equipo]

Prerequisitos: Usuario en la página de registro.

Estado del software: El formulario de registro está disponible.

Datos de prueba:

  • Datos de entrada: Información válida y no válida en el formulario de registro.
  • Datos de salida: Mensajes de error apropiados para entradas inválidas y confirmación de registro para entradas válidas.

Pasos:

  1. Completar el formulario de registro con datos válidos.
  2. Enviar el formulario.
  3. Verificar que se muestra un mensaje de confirmación de registro exitoso.
  4. Completar el formulario con datos inválidos.
  5. Enviar el formulario y verificar que se muestran mensajes de error claros.

Puntos de validación:

  • Verificar que los mensajes de error sean precisos y claros.
  • Asegurarse de que la confirmación de registro se muestre correctamente al ingresar datos válidos.

Identificador: BE-01-2

Nombre corto: Creación de propiedades

Descripción: Verificar que el sistema permita la creación de nuevas propiedades con datos válidos y maneje correctamente las entradas inválidas.

Fecha y autor de creación: [Fecha] - [Tu nombre o equipo]

Prerequisitos: Usuario autenticado con permisos para crear propiedades.

Estado del software: El módulo de creación de propiedades está en funcionamiento.

Datos de prueba:

  • Datos de entrada: Datos válidos e inválidos para la creación de propiedades.
  • Datos de salida: Confirmación de creación o mensajes de error.

Pasos:

  1. Enviar una solicitud para crear una propiedad con datos válidos.
  2. Verificar que el sistema devuelva una confirmación de creación exitosa.
  3. Enviar una solicitud con datos inválidos.
  4. Verificar que el sistema devuelva mensajes de error adecuados.

Puntos de validación:

  • Verificar que las propiedades se creen correctamente en la base de datos con datos válidos.
  • Asegurarse de que los mensajes de error sean precisos y útiles para la corrección de entradas inválidas.

Identificador: FE-02-1

Nombre corto: Filtrado de propiedades por ubicación

Descripción: Verificar que el sistema de filtrado de propiedades por ubicación funcione correctamente en la interfaz de usuario.

Fecha y autor de creación: [Fecha] - [Tu nombre o equipo]

Prerequisitos: Usuario autenticado en la página de búsqueda de propiedades.

Estado del software: El filtro de búsqueda está disponible y funcional.

Datos de prueba:

  • Datos de entrada: Ubicaciones específicas para filtrar propiedades (por ejemplo, "Madrid", "Barcelona").
  • Datos de salida: Lista de propiedades que coinciden con la ubicación seleccionada.

Pasos:

  1. Iniciar sesión y acceder a la página de búsqueda de propiedades.
  2. Introducir una ubicación específica en el filtro de búsqueda.
  3. Aplicar el filtro y revisar los resultados.

Puntos de validación:

  • Verificar que la lista de propiedades mostradas coincide con la ubicación seleccionada.
  • Asegurarse de que no aparezcan propiedades fuera de la ubicación especificada.

Identificador: BE-02-1

Nombre corto: Procesamiento de pagos

Descripción: Verificar que el sistema procesa correctamente los pagos realizados por los arrendadores y actualiza el estado de la solicitud de arriendo.

Fecha y autor de creación: [Fecha] - [Tu nombre o equipo]

Prerequisitos: Usuario autenticado con una solicitud de arriendo en estado "por pagar".

Estado del software: El módulo de procesamiento de pagos está operativo.

Datos de prueba:

  • Datos de entrada: Información de pago válida (monto, banco, número de cuenta).
  • Datos de salida: Confirmación de pago y actualización del estado de la solicitud en la base de datos.

Pasos:

  1. Iniciar sesión y acceder a la página de solicitudes de arriendo.
  2. Seleccionar una solicitud en estado "por pagar" y proceder al pago.
  3. Completar la información de pago y enviar la solicitud.
  4. Verificar que el sistema procese el pago correctamente y actualice el estado de la solicitud a "pagada".

Puntos de validación:

  • Comprobar que la confirmación de pago sea recibida y que el estado de la solicitud se actualice en la base de datos.
  • Verificar que el sistema maneje correctamente errores en el procesamiento del pago y devuelva mensajes de error adecuados si es necesario.
